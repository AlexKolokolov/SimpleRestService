Бета-версия REST сервиса, при обращении к которому по URI localhost:8080/hello/contacts
выдается JSON массив из 100 контактов из базы данных PostgreSQL.
При добавлении к URI параметра nameFilter=regex (regex - регулярное выражение),
массив будет содержать только те контакты из базы данных, в которых поле name не совпадает
с заданным регулярным выражением. Их количество также не будет превышать 100 штук.

***Перед запуском***
В папке src/main/resources находится файл DBConnection.properties, в котором необходимо
перед созданием артефакта изменить значения полей host, port, user, password и т.п. на соответствующие
Вашей базе данных PostgreSQL, в которой содержится таблица contacts, описанная в Вашем задании.
Надеюсь, у Вас есть такая, т.к. у меня нет базы данных с миллионами строк.
Где взять (как создать) такую базу я еще не придумал. Как автоматически поднять такую базу
у Вас - тоже.
На всякий случай в корне проекта лежит бэкап моей небольшой тестовой базы данных(contacts.backup),
из которого можно восстановить базу данных на Вашем сервере PostgreSQL для первичного испытания программы.
После того, как Вы убедились, что у на вашем PostgreSQL сервере есть база данных contacts
с таблицей contacts из двух колонок: id и name, и параметры для подключения к ней прописаны
в DBConnection.properties, можно создать исполняемый jar командой mvn clean package.

***Запуск и останов сервиса***
Программу запускать с помощью start.bat, чтобы при запуске открылось окно консоли,
которое можно будет в последствие закрыть и таким образом остановить работу сервиса.
После запуска сервиса открыть URI localhost:8080/hello/contacts,
либо localhost:8080/hello/contacts?nameFilter=regex, где regex - любое регулярное выражение,
которое будет отфильтровывать данные из базы. Если регулярное выражение окажется невалидным,
оно не будет учтено.
В результате Вы должны увидеть JSON массив из элементов,
включающих в себя два поля: id и name.
